AssessMe User Guide

Panduan penggunaan aplikasi AssessMe

Dokumen ini disusun untuk membantu pengguna memahami alur kerja utama di aplikasi AssessMe, mulai dari masuk ke sistem, membuat assessment, mengisi penilaian, mengelola evidence, melihat laporan, sampai aktivitas administrasi. Struktur manual ini sengaja dibuat ringkas di level penjelasan, tetapi operasional di level langkah.

Cocok untuk Assessor, Manager, Admin, Super Admin, dan Viewer dengan penyesuaian hak akses.
Output manual Dapat dipakai langsung sebagai HTML internal atau dicetak menjadi PDF formal.
Catatan screenshot Template screenshot sudah disiapkan di setiap bagian agar mudah diganti dengan capture aplikasi final.

1. Pendahuluan

Bagian ini menjelaskan konteks penggunaan aplikasi, tujuan manual, gambaran umum modul, serta pembagian hak akses pengguna.

1.1 Latar Belakang dan Tujuan

AssessMe digunakan untuk mendukung proses assessment berbasis COBIT 2019 secara lebih terstruktur, terdokumentasi, dan mudah ditelusuri. Aplikasi membantu tim assessment dalam menyusun objek assessment, memilih GAMO Objective, menetapkan target level, mengisi penilaian, mengunggah evidence, dan menghasilkan laporan yang siap ditinjau.

Manual ini bertujuan agar pengguna dapat menjalankan fungsi aplikasi secara konsisten tanpa harus membaca struktur teknis sistem. Fokus manual adalah penggunaan harian dari sudut pandang user.

1.2 Deskripsi Singkat Aplikasi

Core Module

Assessment Management

Mengelola daftar assessment, detail assessment, wizard pembuatan assessment, dan status pengerjaan.

Core Module

Assessment Execution

Mengisi penilaian per GAMO, melihat target dan current level, menambahkan OFI, serta menyimpan evidence.

Core Module

Reporting

Menampilkan preview laporan, maturity report, gap analysis, summary report, dan ekspor PDF atau Excel.

1.3 Hak Akses Pengguna

Peran Fungsi Utama Cakupan Umum
Viewer Melihat dashboard, daftar assessment, detail assessment, dan laporan sesuai otorisasi. Tidak dapat mengisi jawaban atau mengubah data.
Assessor Membuat assessment, mengisi jawaban, mengunggah evidence, dan melengkapi aktivitas assessment. Fokus pada pelaksanaan assessment.
Manager Memantau assessment, membantu review operasional, dan melihat laporan sesuai kewenangan. Biasanya pada ruang lingkup perusahaan sendiri.
Admin Mengelola user, role tertentu, proses approval tertentu, dan akses administratif. Bergantung konfigurasi permission pada aplikasi.
Super Admin Akses penuh ke administrasi, master data, pengaturan sistem, audit log, dan modul strategis lainnya. Cakupan lintas seluruh data dan seluruh menu.
Tampilnya menu pada sidebar dan hak mengubah data sangat dipengaruhi oleh role dan permission. Karena itu, dua pengguna dapat melihat menu yang berbeda walaupun memakai aplikasi yang sama.

2. Panduan Akses Aplikasi

Bagian ini menjelaskan kebutuhan minimum sisi pengguna, cara mengakses aplikasi, dan langkah masuk ke sistem.

2.1 Spesifikasi Minimum Pengguna

Komponen Rekomendasi Minimum
Perangkat Laptop atau desktop dengan RAM 4 GB atau lebih.
Browser Google Chrome, Microsoft Edge, atau Mozilla Firefox versi terbaru.
Resolusi layar Minimal 1366 x 768 agar tabel dan wizard tampil nyaman.
Koneksi Koneksi jaringan stabil, terutama saat upload evidence dan export laporan.
Pendukung file PDF viewer dan aplikasi office jika hasil export perlu ditinjau offline.

2.2 Akses URL Aplikasi

Buka browser, lalu masukkan URL aplikasi sesuai environment yang dipakai organisasi. Untuk deployment internal, alamat aplikasi dapat berbeda antara server development, staging, dan production.

Contoh format URL: http://alamat-server-anda:port

2.3 Langkah Login

1
Buka halaman login

Pastikan halaman menampilkan field email, password, dan tombol Sign in.

2
Masukkan email dan password

Gunakan akun yang diberikan administrator. Jika browser mendukung, pengguna dapat memakai opsi Remember me.

3
Klik tombol Sign in

Setelah berhasil, sistem akan mengarahkan pengguna ke halaman dashboard.

4
Gunakan fitur lupa password jika diperlukan

Jika pengguna lupa password, gunakan tautan Forgot password? pada form login.

3. Navigasi dan Hak Akses

Berdasarkan implementasi saat ini, menu utama aplikasi berpusat pada Dashboard, Assessments, Reports, Profile, serta Administration atau Master Data untuk role tertentu.

3.1 Menu Utama

Menu

Dashboard

Menampilkan ringkasan KPI, distribusi status assessment, maturity level distribution, tren penyelesaian, dan recent assessments.

Menu

Assessments

Berisi daftar seluruh assessment, pembuatan assessment baru, daftar assessment milik pengguna, serta akses ke detail dan pengisian.

Menu

Reports

Menampilkan daftar assessment yang sudah selesai atau siap dilaporkan, serta fitur preview dan export laporan.

Menu

Profile

Digunakan untuk melihat profil, mengganti password, melihat aktivitas, dan mengatur preferensi user.

Role Based

Administration

Digunakan untuk user management, roles and permissions, audit logs, dan settings sistem sesuai hak akses.

Role Based

Master Data

Digunakan untuk mengelola company, design factors, GAMO objectives, dan questions pada level super admin.

3.2 Prinsip Akses

  • Menu hanya terlihat jika user memiliki role atau permission yang sesuai.
  • Halaman detail assessment dapat bersifat penuh, terbatas, atau read-only sesuai kebijakan otorisasi.
  • Viewer biasanya hanya dapat melihat data dan laporan, tanpa hak menjawab assessment.
  • Assessor dan role di atasnya dapat mengisi jawaban serta mengelola evidence, selama assessment belum ditandai selesai.

4. Panduan Penggunaan Aplikasi

Bagian ini adalah inti manual. Susunan alurnya mengikuti perjalanan user: dari dashboard, membuat assessment, mengisi penilaian, sampai melihat hasil dan tindak lanjutnya.

4.1 Dashboard

Setelah login, pengguna diarahkan ke Dashboard. Halaman ini menampilkan statistik seperti total assessment, jumlah assessment in progress, assessment completed, dan rata-rata maturity level. Selain itu tersedia grafik distribusi status, distribusi level, distribusi kategori GAMO, tren penyelesaian, serta tabel recent assessments.

Untuk pengguna yang memiliki hak membuat assessment, tombol New Assessment tersedia di kanan atas dashboard.
Screenshot SS-04: Dashboard Disarankan menampilkan KPI card dan tabel Recent Assessments dalam satu layar.
Tempel screenshot dashboard di sini.

4.2 Membuat Assessment Baru

Pembuatan assessment dilakukan melalui wizard lima langkah. Tujuannya agar data assessment tersusun rapi sejak awal dan target penilaian per GAMO dapat ditetapkan dengan jelas.

1
Isi Basic Information

Masukkan judul assessment, company, deskripsi, dan periode assessment.

2
Pilih Design Factors

Centang design factor yang relevan dengan konteks organisasi agar assessment lebih terarah.

3
Pilih GAMO Objectives

Pilih objective yang akan dinilai berdasarkan kategori EDM, APO, BAI, DSS, atau MEA.

4
Tetapkan Target Levels

Tentukan target maturity atau capability level untuk tiap objective yang dipilih.

5
Review dan Create

Periksa kembali seluruh input, lalu simpan untuk membentuk assessment baru.

Screenshot SS-05: Wizard Create Assessment Utamakan tampilan step wizard di bagian atas dan form pada step aktif.

4.3 Melihat Detail Assessment

Setelah assessment dibuat, pengguna dapat membuka halaman detail assessment. Di halaman ini tersedia ringkasan informasi dasar, daftar design factors, daftar GAMO objective yang dipilih, target level, hasil assessment per objective, serta action button seperti Answer Assessment, Export PDF, Edit, atau Mark as Complete sesuai hak akses.

4.4 Mengisi Assessment

Pengisian assessment dilakukan melalui halaman Answer Assessment. Halaman ini menggunakan selector GAMO di bagian atas, informasi target dan current level, tombol OFI, dan tab utama untuk aktivitas penilaian.

4.4.1 Memilih GAMO Objective

  • Pilih objective dari dropdown selector di bagian atas halaman.
  • Perhatikan nilai Target dan Current yang ditampilkan.
  • Gunakan tombol Lihat OFI untuk melihat area perbaikan atau catatan OFI.

4.4.2 Tab Level

Tab Level adalah area utama untuk penilaian. Pada area ini user melakukan penilaian aktivitas per level capability, membuka modal penilaian, memberi rating, menambah catatan, dan memantau riwayat perubahan. Jika user tidak memiliki permission menjawab, maka halaman tampil dalam mode read-only.

  • Pilih level yang ingin dinilai.
  • Buka aktivitas yang tersedia.
  • Masukkan rating sesuai hasil assessment.
  • Lengkapi notes bila dibutuhkan.
  • Simpan perubahan.

4.4.3 Tab Repository

Tab Repository digunakan untuk mengelola evidence yang mendukung penilaian. Evidence dapat berupa file atau tautan, tergantung fitur yang diaktifkan pada assessment terkait.

  • Tambahkan evidence untuk aktivitas yang relevan.
  • Gunakan download bila ingin memeriksa ulang dokumen yang sudah tersimpan.
  • Pastikan nama evidence jelas dan konsisten agar mudah direview.

4.4.4 Tab Summary All GAMO Objectives

Tab Summary menampilkan ringkasan lintas objective, sehingga pengguna dapat meninjau progres secara menyeluruh tanpa membuka satu per satu objective.

Jika assessment sudah berstatus completed, sistem dapat membatasi perubahan jawaban. Pastikan semua evidence dan penilaian telah diperiksa sebelum menandai assessment selesai.

4.5 Menyelesaikan Assessment

Setelah seluruh aktivitas terisi dan evidence telah diunggah, pengguna yang berwenang dapat menandai assessment sebagai selesai melalui tombol Mark as Complete pada halaman detail assessment.

  • Pastikan seluruh objective yang dipilih telah memiliki jawaban yang memadai.
  • Pastikan evidence utama telah terunggah.
  • Pastikan ringkasan summary menunjukkan progres yang sesuai.
  • Baru setelah itu lakukan finalisasi status.

Jenis Laporan

Export

Format Keluaran

PDF untuk distribusi formal, dan Excel untuk analisis detail atau pengolahan lanjutan.

4.7 Recommendations dan Action Plans

Setelah assessment selesai, organisasi dapat memanfaatkan modul Recommendations dan Action Plans untuk menindaklanjuti gap yang ditemukan. Recommendations dapat dibuat manual atau dihasilkan otomatis, kemudian diteruskan ke action plan untuk pelacakan progres.

  • Buka halaman Recommendations dari assessment terkait.
  • Gunakan tombol Auto-Generate bila ingin menghasilkan rekomendasi berbasis gap analysis.
  • Tentukan prioritas, penanggung jawab, target date, dan status progres.
  • Gunakan Action Plan Dashboard, Timeline View, atau Progress Tracking untuk monitoring pelaksanaan.

5. Panduan Admin dan Super Admin

Bagian ini dapat dipakai sebagai bab terpisah untuk pengguna administratif. Beberapa menu hanya muncul untuk role tertentu.

5.1 User Management

Admin atau Super Admin dapat mengelola data pengguna, termasuk membuat user baru, mengubah data user, menonaktifkan akun, dan menyesuaikan role sesuai kebutuhan operasional organisasi.

5.2 Roles and Permissions

Menu ini dipakai untuk mengatur model hak akses. Pastikan perubahan permission mengikuti kebijakan organisasi, karena perubahan di sini akan berdampak langsung pada menu yang terlihat dan aksi yang diizinkan di aplikasi.

5.3 Audit Logs

Audit log membantu administrator menelusuri aktivitas sistem. Modul ini bermanfaat untuk monitoring perubahan, penelusuran kejadian, dan kebutuhan audit internal.

5.4 System Settings

Gunakan menu settings untuk pengaturan global aplikasi bila modul tersebut diaktifkan untuk role Anda. Perubahan di area ini sebaiknya dibatasi pada petugas yang memahami dampak konfigurasi sistem.

5.5 Master Data

Pada level super admin, master data digunakan untuk menjaga kualitas referensi sistem, seperti daftar company, design factor, GAMO objective, dan question atau activity yang menjadi dasar assessment.

Sebelum mengubah master data, disarankan memastikan bahwa perubahan tidak mengganggu assessment yang sedang berjalan, terutama jika terkait objective, question, atau struktur referensi yang sudah dipakai pada assessment aktif.

6. Troubleshooting

Daftar masalah umum berikut dapat dimasukkan ke manual agar tim pengguna tidak selalu harus menghubungi admin untuk masalah dasar.

Masalah Penyebab Umum Tindakan
Tidak bisa login Email atau password salah, atau akun belum aktif. Periksa credential, gunakan fitur lupa password, atau hubungi admin.
Menu tertentu tidak muncul Role atau permission tidak sesuai. Pastikan user telah diberi role yang benar oleh admin.
Tidak bisa mengubah assessment Assessment berstatus completed atau user berada dalam mode read-only. Periksa status assessment dan hak akses user.
Evidence gagal terunggah Koneksi tidak stabil, file terlalu besar, atau format file tidak sesuai. Ulangi upload dengan koneksi stabil dan gunakan file yang sesuai kebijakan organisasi.
Export report tidak keluar Assessment belum siap dilaporkan atau browser memblokir download. Pastikan assessment memenuhi syarat pelaporan dan cek pengaturan download browser.

Kontak Dukungan Internal

Tambahkan informasi PIC atau helpdesk organisasi pada bagian ini sebelum manual didistribusikan secara resmi.

Contoh: Helpdesk TI, admin aplikasi, nomor ekstensi, email support, atau SLA penanganan insiden.
